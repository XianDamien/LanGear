# PRD（草案）：多语言复述训练与AI评分

## 0. 文档信息
- 版本：v0.1（待确认）
- 读者：开发
- 默认技术栈：`Langear/v0`（Next.js）+ `Langear/backend`（FastAPI）+ `Langear/RetellingASR`（AI评测）
- 默认模型：Gemini

## 1. 背景与目标
当前用户系统无法稳定区分与记录练习历史，主要原因是“应用数据数据库”和“AI分析结果数据库”分离，导致练习记录 ID 不一致。目标是：
- 统一为**单一数据库**作为唯一事实源。
- **ASR 状态由后端持有**（内存/Redis），数据库仅存稳定数据与最终结果。
- 提供可直接使用与导入练习音频的 Web 版训练流程，并加入 AI 评分闭环。

## 2. 范围（MVP 必做 / 迭代）
### MVP 必做
- 登录与语言对选择（邮箱+密码；英-中、法-中等）。
- 页面一：数据看板（当前牌组、今日任务、连续练习日历、收藏卡片入口）。
- 页面二：牌组资源库（source-单元-课级联结构，source=书+册合并；内置牌组如新概念2、雅思听力）。
- 页面三：导入自定义牌组（音频上传、AI切分+人工校对、生成句子/段落卡）。
- 核心卡片训练：正面练习（听-复述-录音）、背面复盘（原文/用户音频+转写+高亮）。
- 单句 AI 反馈、整课 AI 汇总、FSRS 复习调度（基础接入）。
- 音频持久化到 OSS。

### 暂不做（可延后）
- 复杂权限体系（教师/班级）
- 社交与排行榜
- 付费体系

## 3. 用户流程（摘要）
1) 登录 → 选择语言对  
2) 看板 → 今日任务 / 连续天数 / 收藏卡片快速练习  
3) 资源库 → 选择 source-单元-课 → 开始练习  
4) 卡片正面：反复播放原音频 → 录音（流式 ASR 反馈）  
5) 翻面：展示原文音频+文本、用户音频+转写、名词短语高亮、动词下划线  
6) 自评难度（FSRS）+ 记录笔记  
7) 单句 AI 反馈异步生成；完成一课触发全局总结  
8) 自定义牌组：上传音频 → 选择模型/难度/时长 → AI切分 → 人工确认 → 发布  

## 4. 核心功能细化
### 4.1 数据看板
- 今日新学/复习任务、连续学习天数、收藏卡片入口。
- 数据来源：用户进度表 + 练习记录表（FSRS 全局调度，不按牌组单独配置）。

### 4.2 牌组资源库
- 内容结构：source（书+册合并）→ 单元 → 课。
- 每张卡可包含：原文、译文、音频、解析、笔记字段。

### 4.3 自定义牌组
- 音频上传后进入“切分任务”队列。
- 支持句子/段落两种卡片类型。
- 允许手工调整切分边界与文本。

### 4.4 训练与复盘
- 正面：播放原音频、多次录音、录音完成后延迟显示答案（可配置）。
- 背面：原文/用户音频与文本，词性标注（名词短语高亮、动词下划线）。
- 词典与 AI 解释：支持内置词典/导入词典。

### 4.5 AI 评测与总结
- 单句反馈：翻面时上传“原音频+练习音频”，AI 返回结构化反馈。
- 课级总结：聚合单句反馈，输出模式分析与重点建议。
- 模型默认 Gemini，可扩展多模型。

## 6. 埋点/指标（最小集）
- DAU、连续练习天数、单卡完成率、AI 反馈可用率、课级总结完成率。

## 7. Open Questions

- 词典来源与授权（如需公开上线）。
- ASR 状态存储选择：内存 vs Redis（多实例需 Redis）。
