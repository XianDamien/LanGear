# PRD（MVP 版）：复述训练与 AI 反馈闭环

## 1. 文档信息
本文件用于定义复述训练产品的最小可行版本。该版本的目标是保证训练闭环的完整性与可验证性，而不是功能覆盖面。所有关键流程必须成功后才能进入下一步，开发阶段不允许以降级或替代方案绕过失败。

## 2. 产品目标与范围
本产品面向口语复述训练场景，目标是建立“听、复述、AI 反馈、复盘、复习”的完整闭环。MVP 阶段只关注内置教材的训练流程，不包含自定义上传与字幕切分等扩展功能。内置教材至少包含新概念英语与剑桥雅思听力，后续教材可以通过数据导入扩展。

MVP 需要满足以下结果：
1. 用户可以选择教材与课文并完成全部卡片训练。
2. 每张卡片都能生成 AI 单句反馈并持久化记录。
3. 当课文全部完成时自动生成课级总结并持久化记录。
4. FSRS 调度必须生效，并且在同一教材来源内独立运作。

## 3. 用户与权限
MVP 采用管理员单用户模式。系统仅维护一个管理员账号，所有数据属于该账号。当前阶段不提供访客模式与登录注册流程。

## 4. 核心用户流程
1. 管理员进入应用并在教材库中选择教材来源与课文。
2. 系统加载课文中的卡片列表并进入卡片训练流程。
3. 正面界面播放原音频，管理员录音复述。
4. 翻面后展示原文音频、原文文本、用户录音、转写文本与词块对齐结果，生成 AI 单句反馈并写入训练记录
5. 管理员选择选择相应的评分按钮（重来、困难、良好、简单）
6. 当课文的全部卡片都完成后，系统自动生成课级总结并写入训练记录。
7. 课程总结页面展示本课文的整体问题模式与学习建议。

所有步骤中任何一个关键动作失败都必须阻断流程并提示错误，例如音频上传失败、AI 反馈生成失败、课程总结生成失败。

## 5. 前端模块划分（以 frontend 为基准）
### 5.1 Dashboard
Dashboard 负责展示学习概览与入口。它需要允许管理员创建筛选牌组，需要设定每日新学数量、每日复习数量与来源教材范围，并显示连续学习天数与热力图。它还需要提供“继续上次练习”/“继续特定牌组的练习”的快捷入口。

### 5.2 Library
Library 负责展示教材层级结构。层级顺序为教材来源、单元、课文。用户在课文级别进入训练流程。

### 5.3 Study Session
Study Session 负责卡片训练流程。正面展示原音频播放与录音按钮，背面展示原文、译文、用户录音、转写文本与词块对齐结果，并提供评分入口。

### 5.4 Card Detail
Card Detail 用于查看单句反馈的详细内容，包括发音准确度、内容完整度与流畅度等关键指标。

### 5.5 Summary
Summary 用于展示课级总结与模式归纳，强调本课文的共性问题与优先改进建议。

### 5.6 Settings
Settings 提供管理员数据清理与导出功能，用于开发阶段的快速验证与数据回滚。

## 6. 后端模块划分
后端模块按职责划分为内容管理、训练记录、AI 评测和 FSRS 调度。
1. 内容管理负责维护教材来源、单元、课文与卡片数据。
2. 训练记录负责写入评分结果与 AI 反馈。
3. AI 评测负责生成单句反馈与课级总结。
4. FSRS 调度负责在教材来源内部独立更新复习状态。

## 7. 数据库设计（6 张表）
数据库采用 SQLite 单库。为了减少冗余，教材层级使用单表表达，卡片作为独立表保存训练最小单元。

### 7.1 users
用于存储管理员账号信息。仅维护一个账号。

### 7.2 decks
统一表示教材来源、单元与课文三层结构。通过 parent_id 构成树形结构。每个节点包含标题与排序字段。

### 7.3 cards
用于存储最小训练单元。每条记录关联到课文级 deck 节点。字段包含正面文本、背面文本与音频路径。

### 7.4 user_card_srs
用于存储 FSRS 调度状态。该表以卡片为单位记录状态、稳定度、难度与下次复习时间。FSRS 调度按教材来源维度独立。

### 7.5 review_log
用于存储训练记录与 AI 反馈。每条记录包含卡片标识、评分结果与 AI 反馈 JSON。单句反馈与课级总结都写入该表，但通过记录类型区分。

### 7.6 settings
用于存储系统级配置项，例如每日任务数量与默认教材范围。

## 8. API 设计
后端提供以下接口以支持核心流程。接口不提供降级路径，所有异常都必须返回明确错误并阻断流程。
1. 获取教材层级结构的接口，用于构建 Library 页面。
2. 获取课文卡片列表的接口，用于进入训练。
3. 提交复述与评分的接口，负责生成 AI 单句反馈并写入 review_log。
4. 获取课文总结的接口，课文完成后自动触发并写入 review_log。
5. 获取 Dashboard 统计数据的接口，用于展示任务与热力图。

## 9. AI 反馈规范
单句反馈必须包含发音准确度、内容完整度与流畅度三大核心指标，并输出统一的结构化字段。课级总结必须聚合单句反馈并给出共性问题与优先建议。所有 AI 输出字段固定，避免前端无法稳定渲染。

## 10. 错误处理与无降级原则
MVP 阶段所有关键流程必须严格成功，不能使用备用流程完成任务。示例包括：
1. 音频上传失败时必须阻止评分提交。
2. AI 单句反馈失败时必须阻止卡片完成。
3. 课级总结生成失败时必须阻止课文完成。

## 11. 验收标准
1. 新概念英语与剑桥雅思课文可以完整完成训练闭环。
2. 单句反馈必须生成并写入 review_log。
3. 课级总结必须自动生成并写入 review_log。
4. FSRS 调度必须按教材来源维度独立更新。
5. Dashboard 能正确显示任务配置、连续天数与热力图。
